# test_plan.md

## 1. 目的
- XIAO ESP32C3を3台使い、P2P通信・中継・距離拡張の実機成立性を段階確認する。

## 2. テスト体制
- NodeA: 送信起点（ゲートウェイ候補）
- NodeB: 中継（リレー）
- NodeC: 受信終端（エッジ）
- 試験時の固定情報:
  - ポート割当
  - ファームウェアバージョン
  - 電源方式（USB/電池）
  - アンテナ種別

## 3. 事前準備
1. `tools/prepare_test_session.ps1 -Ports COM5,COM6,COM7` を実行
2. `tools/flash_all.ps1 -Ports COM5,COM6,COM7` で同一ビルドを書き込み
3. `tools/monitor_all.ps1 -Ports COM5,COM6,COM7` で3台ログ監視を開始
4. 開始時刻と環境条件（場所、障害物、チャンネル）を記録

## 4. テスト項目一覧

| ID | 項目 | 目的 | 合格条件 |
|---|---|---|---|
| T01 | 起動・識別確認 | 3台の識別とログの分離確認 | 全ノードが起動ログを出し識別可能 |
| T02 | 1ホップP2P | NodeA→NodeBの疎通確認 | 送信100回で受信成功率95%以上 |
| T03 | 逆方向P2P | NodeB→NodeAの疎通確認 | T02同等の成功率 |
| T04 | 2ホップ中継 | NodeA→NodeB→NodeC成立確認 | NodeCで受信しHop=2が記録される |
| T05 | 重複排除 | 同一Packet再受信時の抑止確認 | NodeCが重複を1回のみ採用 |
| T06 | Ping応答 | Node間RTT測定と応答確認 | Ping 100回で応答率95%以上 |
| T07 | 障害物試験 | 屋内50〜100m相当条件評価 | 目標PDRを満たす設定が1つ以上見つかる |
| T08 | 見通し距離試験 | 長距離の限界評価 | 距離別PDR/遅延を取得できる |
| T09 | 再起動復帰 | 運用回復性確認 | 任意ノード再起動後に通信再開する |
| T10 | 長時間安定性 | 連続運用耐性確認 | 60分以上でハング/再起動なし |

## 5. 各テストの実施要領

### T01 起動・識別確認
- 手順:
  - 3台を起動し、ノードIDとポート対応をログで確認
- 記録:
  - 起動時刻、ノードID、ファームバージョン

### T02/T03 1ホップP2P
- 手順:
  - 送信側で100パケット連続送信
  - 受信側で受信数、欠落数、RSSIを記録
- 記録:
  - 送信数、受信数、PDR、平均RSSI、最大遅延

### T04 2ホップ中継
- 手順:
  - NodeBを中継ノードとして配置
  - NodeAからNodeC宛に送信
- 記録:
  - NodeB転送回数、NodeC受信数、Hop情報

### T05 重複排除
- 手順:
  - 同一 `src_id + seq` を複数回送信
- 記録:
  - 採用件数、破棄件数、誤採用の有無

### T06 Ping応答
- 手順:
  - NodeAからNodeB/NodeCへPingを連続送信
  - RTTと応答率を記録
- 記録:
  - 送信数、応答数、PDR、平均/最大RTT

### T07 障害物試験
- 手順:
  - 壁あり区間や人体遮蔽を含む経路で測定
  - 位置を変えて複数ケース測定
- 記録:
  - 距離、障害物条件、PDR、遅延、再送率

### T08 見通し距離試験
- 手順:
  - 10m, 30m, 50m, 100m ... のように段階測定
- 記録:
  - 各距離でのPDR、平均遅延、RSSI

### T09 再起動復帰
- 手順:
  - 通信中に任意ノードを再起動
  - 復帰までの時間を計測
- 記録:
  - 復帰時間、再参加失敗の有無

### T10 長時間安定性
- 手順:
  - 定周期送信を60分以上継続
- 記録:
  - エラー回数、再起動回数、温度関連イベント

## 6. 判定基準（初期）
- 必須合格:
  - T01〜T06
- 目標合格:
  - T07〜T10
- 総合判定:
  - 必須合格が全て成立し、目標項目で1つ以上の有効設定が得られること

## 7. 記録テンプレート
- セッション単位で以下を保存:
  - `test_logs/<timestamp>/session.md`
  - シリアルログ（NodeA/NodeB/NodeC）
  - 失敗時の再現手順
