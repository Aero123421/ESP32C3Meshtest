# AGENTS.md

## 1. プロジェクトの目的
- 対象ハード: Seeed Studio XIAO ESP32C3 を3台使用
- 目的: 「P2P通信の確立」→「メッシュ化」→「通信距離の拡張」を段階的に実証する
- 成果物:
  - 実機で再現できる書き込み・監視・試験手順
  - Wi-Fi/BLEメッシュ設計ドキュメント
  - 長距離化に向けた設計上の注意点と試験結果

## 2. 推奨ディレクトリ構成
- `platformio.ini`, `src/`, `include/`: ESP32-C3向けファームウェア
- `pc_app/`: PC側補助アプリ
- `docs/`: 設計・調査・手順・試験計画
- `tools/`: 運用補助スクリプト（PowerShell）
- `test_logs/`: 実機試験ログ（自動/手動で保存）

## 3. 開発ルール
- 変更前提:
  - 小さく変更し、実機で確認し、記録を残す
  - 仕様判断は `docs/` に根拠付きで記録する
- コード運用:
  - `src/` `include/` `pc_app/` は担当者以外が不用意に変更しない
  - 共通設定や手順変更は先に `docs/` と `tools/` を更新する
- 無線設計:
  - 2.4GHz帯は干渉影響が大きいため、チャンネル固定で試験する
  - 長距離試験時は外部アンテナを使用し、アンテナ向きと高さを記録する
- 安全・法規:
  - 使用地域の電波法規（日本: 技適、米国: FCC）に適合した範囲で運用する
  - 高負荷連続送信時は発熱に注意し、筐体内の放熱を確保する

## 4. 実機テスト手順（3台）
1. 事前準備
   - 3台のポートを固定割当（例: NodeA=`COM5`, NodeB=`COM6`, NodeC=`COM7`）
   - 外部アンテナ接続、電源ケーブル（データ対応）確認
2. セッション初期化
   - `tools/prepare_test_session.ps1` を実行し、記録テンプレートを生成
3. 一括書き込み
   - `tools/flash_all.ps1 -Ports COM5,COM6,COM7` を実行
4. 同時モニタ
   - `tools/monitor_all.ps1 -Ports COM5,COM6,COM7` を実行
5. 試験実施
   - `docs/test_plan.md` の順で実施し、結果を `test_logs/` に記録
6. 片付け
   - 失敗ケースと再現条件を残し、再試験条件を明示する

## 5. ドキュメント運用ルール
